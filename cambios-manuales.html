Amazon S3

S3.1: Amazon S3 Bucket Server-Side Encryption Disabled on Sensitive Buckets ⭐:

Esta incidencia indica que el cifrado del lado del servidor no está habilitado en los buckets de Amazon S3 que contienen datos sensibles. Esto puede dejar la información vulnerable en caso de una violación de seguridad.

Resolución:

Habilita el cifrado del lado del servidor (SSE) para los buckets de Amazon S3 que almacenan datos sensibles. Puedes hacer esto utilizando el cifrado de Amazon S3, que es la opción predeterminada. También puedes optar por utilizar el cifrado de AWS Key Management Service (KMS) para un control más granular sobre las claves de cifrado.

  2. Nos marcamos el check all public access:

  3. confirmamos.

  4. Solucionado.

S3.9: Amazon S3 Bucket Policy does not require secure connections (TLS) ⭐⭐:

S3.9 y S3.13 se soluciona también agregando una Bucket policy

Lo mejor para solucionar ambas incidencias es poner ambas Bucket policy juntas.

Esta incidencia indica que las políticas del bucket de Amazon S3 no requieren conexiones seguras (TLS). Esto podría dejar expuesta la comunicación entre los clientes y el bucket a ataques de tipo "man-in-the-middle".

Resolución:

Actualiza las políticas del bucket de Amazon S3 para requerir conexiones seguras mediante TLS (Transport Layer Security). Esto se puede lograr configurando la política de cubo para denegar todas las solicitudes HTTP y permitir solo solicitudes HTTPS. Asegúrate de que tus aplicaciones y servicios que acceden al bucket estén configurados para utilizar conexiones seguras.

  1. Nos dirigimos al servicio de S3:

  2. Elegimos el bucket que tiene el hallazgo

  3. Nos dirigimos a la pestaña Permissions y damos clic en edit de Bucket policy.

  4. ahora pegamos.

    { "Version": "2008-10-17", "Statement": [ { "Sid": "DevSecOps-ServerAccessLogs-ForceSSLOnlyAccess", "Effect": "Deny", "Principal": { "AWS": "*" }, "Action": "s3:*", "Resource": [ "arn:aws:s3:::{Nombre del Bucket}/*", "arn:aws:s3:::{Nombre del Bucket}" ], "Condition": { "Bool": { "aws:SecureTransport": "false" } } } ] }

S3.13: Amazon S3 Bucket Logging Not Enabled ⭐⭐:

S3.9 y S3.13 se soluciona también agregando una Bucket policy

Lo mejor para solucionar ambas incidencias es poner ambas Bucket policy juntas.

Esta incidencia indica que el registro de acceso al bucket de Amazon S3 no está habilitado. Sin registros, no hay una forma efectiva de rastrear las actividades en el bucket, lo que dificulta la detección de actividades maliciosas o la realización de auditorías.

Resolución:

Habilita el registro de acceso al bucket de Amazon S3. Puedes hacer esto configurando un bucket de destino para almacenar los registros de acceso. Considera la activación de la opción de registro de acceso en todas las regiones donde tengas buckets relevantes. Desarrolla políticas y procedimientos para monitorear y analizar los registros de acceso regularmente, lo que te permitirá identificar y responder a posibles amenazas de seguridad. En resumen, para resolver estas incidencias, es crucial implementar medidas de seguridad recomendadas, como el cifrado del lado del servidor, la habilitación de conexiones seguras mediante TLS y la activación del registro de acceso en los buckets de Amazon S3. Estas acciones ayudarán a fortalecer la seguridad de tus datos almacenados en la nube y reducirán el riesgo de exposición a amenazas de seguridad.

  2. Elegimos el bucket que tiene el hallazgo

  3. Nos dirigimos a la pestaña Permissions y damos clic en edit de Bucket policy.

  4. ahora pegamos.

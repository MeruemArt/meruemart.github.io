Layers en AWS Lambda

Descripción General

Con la actualización a la versión 6.5.0, el pipeline de despliegue de microservicios en Python en Azure DevOps ahora se encarga de crear layers de Python que se utilizan en los servicios de AWS Lambda. Esta mejora optimiza tanto el proceso de despliegue como la gestión de dependencias.

Con el nuevo mecanismo, no se verá afectado lo que ya está en funcionamiento, ya que este solo operará si la carpeta dentro de layers contiene un archivo requirements.txt.

Integración en el Pipeline

Ubicación en el Pipeline

El script se ejecuta durante la etapa de Build en el pipeline de microservicios Python, específicamente después de realizar el SAM Build.

Parámetros Requeridos

El script requiere los siguientes parámetros que son pasados desde el template principal:

parameters: - name: WORKSPACE default: '.' - name: PYTHON_VERSION - name: PYTHON_PLATFORM

Configuración en el Template Principal

En pythonMicroserviceTemplate.yml

Los parámetros se definen en la sección de parámetros del template:

parameters: - name: PYTHON_VERSION displayName: Versión de Python default: '3.11' values: - '3.13' - '3.12' - '3.11' - '3.10' - name: PYTHON_PLATFORM displayName: Arquitectura de lambdas python values: - 'x86_64 / amd64' - 'aarch64 / arm64' default: 'aarch64 / arm64'

En build.yml

El script se invoca con los siguientes parámetros:

- template: ../../../steps/script/scriptUVPythonLayer.yml@templates parameters: WORKSPACE: "$(Build.SourcesDirectory)/$(Build.Repository.Name)" PYTHON_VERSION: ${{ parameters.PYTHON_VERSION }} PYTHON_PLATFORM: ${{ parameters.PYTHON_PLATFORM }}

Flujo de Ejecución

Preparación del Ambiente:

  • El script se ejecuta en un contenedor Linux con AWS CLI

  • Se establece el directorio de trabajo

  • Se configuran las variables de entorno necesarias

Procesamiento de Layers:

  1. Busca directorios dentro de la carpeta layers

  2. Para cada directorio que contiene un requirements.txt:

    • Instala las dependencias usando uv pip

    • Crea un archivo ZIP con las dependencias

    • Renombra el ZIP según el nombre del directorio

    • Limpia los archivos temporales

Consideraciones Importantes

Versión de Python:

  • Debe coincidir con la versión utilizada en el microservicio

  • Se recomienda usar la misma versión en desarrollo y producción

Plataforma:

  • aarch64 / arm64 para arquitectura ARM64

  • x86_64 / amd64 para arquitectura x86_64

  • La elección afecta la compatibilidad con AWS Lambda

Estructura de Directorios:

  • Los layers deben estar en el directorio layers/

  • Cada layer debe tener su propio requirements.txt

Ejemplo de Uso

Para implementar un nuevo layer:

  1. Crear un directorio en layers/:

layers/nombrelayer

  1. Crear el archivo requirements.txt con el contenido de la dependencia:

requests==2.31.0

El pipeline automáticamente:

  • Instalará las dependencias

  • Creará el ZIP

  • Lo nombrará como nombrelayer-python-layer.zip

Mejores Prácticas

Mantenimiento de Dependencias:

  • Especificar versiones exactas en requirements.txt

  • Actualizar regularmente las dependencias

  • Mantener los layers lo más pequeños posible

Optimización:

  • Usar --no-installer-metadata y --no-compile-bytecode para reducir el tamaño

  • Considerar la arquitectura de destino al elegir PYTHON_PLATFORM

Seguridad:

  • Revisar regularmente las dependencias en busca de vulnerabilidades

  • Mantener las versiones de Python actualizadas

Solución de Problemas Comunes

Error de Instalación:

  • Verificar que la versión de Python sea compatible

  • Comprobar que las dependencias existen en PyPI

Problemas de Tamaño:

  • Reducir el número de dependencias

  • Usar versiones más ligeras de las bibliotecas

Errores de Compatibilidad:

  • Asegurar que la plataforma seleccionada coincide con el entorno de ejecución

  • Verificar que las dependencias son compatibles con la arquitectura elegida
